# 远端存储格式 v1

任务在 WebDAV备份根下使用明文任务名作为物理目录。任务名可通过服务端 `MOVE` 修改，永久身份由随机任务 UUID 决定。

```text
任务名/
  task-a.json
  task-b.json
  catalog-a.wci
  catalog-b.wci
  objects/
    3f/
      <随机文件组ID>/
        part-00001-<随机对象ID>.wcb
        part-00002-<随机对象ID>.wcb
  snapshots/
    <UTC时间-随机后缀>/
      index-a.wci
      index-b.wci
      COMPLETE
      KEEP                 # 可选，非空JSON
```

`task-a.json` 和 `task-b.json` 是相同语义的双份公开启动描述，包含格式版本、任务UUID、任务名、模式、Argon2id盐及创建/更新时间，不含任务密码、源路径或文件名。

`catalog-*.wci` 和 `index-*.wci` 使用与数据对象相同的加密容器。两个副本使用不同随机nonce；读取时接受任一有效副本并选择较新的目录状态。

## 对象容量与装包

用户按任务固定选择最终密文对象上限：

- 1,000,000,000字节
- 2,000,000,000字节
- 3,700,000,000字节

实现为加密头、认证标签和索引元数据预留16 MiB。大于有效容量80%的文件独立存放；超过容量时连续切分，最后尾块不会与其他文件混装。其他新增文件按源根、目录及文件名排序后依次装包，单包最多5000个文件。

同一文件的所有 `part-*` 位于同一个随机文件组目录。随机对象ID避免泄露原文件名；`part-00001` 仅暴露顺序。

## 流式加密容器

所有 `.wcb` 和 `.wci` 使用版本化容器：

```text
8 bytes   magic = WCBLOCK1
4 bytes   大端序公开头JSON长度
N bytes   公开头JSON
重复：
  4 bytes 大端序密文帧长度
  N bytes XChaCha20-Poly1305密文与认证标签
4 bytes   零长度结束标记
```

公开头包含格式版本、任务UUID、随机对象ID、对象类型、Argon2id参数和盐、随机16字节nonce前缀，以及4 MiB帧容量。

任务密钥通过 Argon2id 得到：

- 时间成本：3
- 内存：64 MiB
- 并行度：2
- 输出：256 bit

每个加密帧使用 XChaCha20-Poly1305。24字节nonce由随机16字节前缀和大端序64位帧序号组成。关联数据为公开头 SHA-256 加帧序号。应用没有自制加密算法。

首个加密明文内容为元数据JSON长度和元数据JSON；随后是文件数据。数据对象元数据记录：

- 任务、对象和文件组ID
- 分块序号与总块数
- 文件ID、首次路径、大小、SHA-256、创建/修改时间
- 文件在对象内的偏移与长度
- 大文件片段在原文件中的偏移

因此在索引完全丢失但所有对象仍存在时，可用任务密码扫描每个对象并按首次路径重建。若唯一对象被整个删除，或文件后来移动但索引丢失，无法证明完整性或恢复最新路径。

## 发布与完整性

对象先上传到带随机 `.partial-*` 后缀的临时路径。`PROPFIND` 返回的长度匹配后，用 WebDAV `MOVE` 发布正式随机名。失败对象删除并从头重传。

快速完整性检查只读取远端名称和大小，不下载内容，也不将 ETag 当作内容哈希。恢复时每个加密帧必须通过AEAD认证，原文件落盘后再比较备份索引中的 SHA-256。

数据对象不做副本或纠删码冗余。双份仅用于很小但关键的任务与版本索引。
