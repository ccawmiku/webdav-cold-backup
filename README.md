# WebDAV Cold Backup

面向 NAS 冷备份和异地转移的 WebDAV 分块备份工具。服务端以 Docker Compose 部署，提供简体中文管理后台；另有无需安装、无需 Docker 的 Windows 10/11 x64 离线恢复程序。

> 当前格式不提供数据冗余。任何永久丢失的数据块都会导致对应大文件或小文件包无法完整恢复，但不会影响其他对象。任务密码丢失后无法恢复数据。

## 主要能力

- 两种固定任务模式：最近 N 个快照，以及只保留首次路径的持续上传归档。
- 原文件超过对象容量时独立切分；大于容量 80% 的文件独立存放；小文件按相邻目录优先装包。
- 最终远端对象上限固定为 1,000,000,000、2,000,000,000 或 3,700,000,000 字节。
- Argon2id 派生任务密钥，XChaCha20-Poly1305 独立认证每个 4 MiB 加密帧。
- 每个对象包含加密的自恢复头；每个任务和快照保存双份索引。
- 快照跨版本复用未变化、移动或重命名文件的数据；同版本复制仍作为新文件上传。
- 归档只按内容首次出现上传，后续移动、重命名和复制不会改变首次记录。
- WebDAV 临时对象上传、大小确认后 `MOVE` 发布；中断对象从头重传。
- 一次快速重试，再按 1、5、25 分钟进行三次指数退避。
- 多任务串行队列，任务内部全局 1—3 个对象并发；上传和下载分别限速。
- 手动、每天 `HH:mm`、每周指定星期和时间三种计划；错过的计划不补执行。
- 远端只按随机对象名和大小检查，不下载数据；未引用对象必须输入任务密码后手动清理。
- NAS 后台可从 WebDAV直接恢复，也可导出 `.backup-plan`，手动下载所需块后从映射的完整任务目录恢复。
- Windows便携程序只读取已完整下载的任务目录，默认选择最新完整版本。
- 本地数据库完全丢失时可从远端重建；远端索引也丢失时可扫描对象自恢复头进行尽力抢救。

## 部署

创建本地目录并确保容器用户有读写权限：

```text
data/
  config/   SQLite、本地索引和运行记录
  cache/    临时加密对象，最大约为块容量 × 上传并发
  sources/  备份源，只读映射
  restore/  NAS恢复输出
```

复制 `docker-compose.yml`，按 NAS 实际路径设置环境变量后启动：

```bash
WCB_UID=1026 WCB_GID=100 \
WCB_SOURCE_PATH=/volume1/media \
WCB_RESTORE_PATH=/volume1/restore \
docker compose up -d
```

默认界面地址为 `http://NAS_IP:8080`。镜像仅发布明确版本：

```text
ghcr.io/ccawmiku/webdav-cold-backup:v1.0.0
```

不会发布 `latest` 标签。服务无登录鉴权，只应在可信局域网开放。

### Compose变量

| 变量                  | 默认值           | 说明                                                       |
| --------------------- | ---------------- | ---------------------------------------------------------- |
| `WCB_UID` / `WCB_GID` | `65532`          | 容器运行用户，需能写入 config/cache/restore 并读取 sources |
| `WCB_PORT`            | `8080`           | 管理后台端口                                               |
| `WCB_CONFIG_PATH`     | `./data/config`  | SQLite和本地索引目录                                       |
| `WCB_CACHE_PATH`      | `./data/cache`   | 临时加密对象目录                                           |
| `WCB_SOURCE_PATH`     | `./data/sources` | NAS备份源，只读挂载                                        |
| `WCB_RESTORE_PATH`    | `./data/restore` | NAS恢复目标根                                              |

时区、并发和限速在网页“全局设置”中配置，默认时区为 `Asia/Singapore`。

## 首次备份

1. 在网页创建任务，选择快照或归档模式。模式、任务密码和块容量创建后不能修改。
2. 添加一个或多个不互相嵌套的映射源目录；恢复根名称重复时会自动添加数字后缀。
3. 填写 WebDAV 地址、备份根、用户名和密码。程序会用小型临时对象测试 `MKCOL`、`PUT`、`PROPFIND`、`MOVE` 和 `DELETE`。
4. 选择手动、每天或每周计划。计划触发时只执行一次；上次仍未结束则跳过本次触发。
5. 任务创建后可立即运行。10分钟内发生修改的文件静默跳过；符号链接、回收站和 NAS 系统目录会忽略并写入运行摘要。

支持的默认排除目录包括 `#recycle`、`@Recycle`、`$RECYCLE.BIN`、`.Trash` 和 `@eaDir`。源根不可访问或任何子目录权限不足时，本次扫描停止且不发布新版本。

## 快照与归档语义

### 快照模式

- 每次有变化的成功扫描发布一个不可变版本；完全无变化时只记录运行结果。
- 源端删除只影响新版本。只要旧版本仍保留，数据块就不会清理。
- 默认保留最近3个未锁定版本；永久版本额外保留，只能先解锁再删除。
- 单文件上传失败仍会发布明确标记为“不完整”的版本，且计入保留数量。
- 不提供历史版本内的单文件删除，只能删除整个版本。

### 归档模式

- 每个内容哈希仅在首次出现时上传并记录当时路径。
- 后续移动、改名、复制和本地删除均不改变已归档记录。
- 后台可手动删除文件记录；包内仍有其他文件时不会重写或释放整个包。
- 被手动删除的源文件若仍在 NAS，下次扫描会重新加入，不提供永久排除规则。

## 恢复

恢复前仅用索引检查所有所需随机对象名和大小。具备全部块的文件可以继续恢复，缺块文件单独排除。

- 恢复完成后计算原文件 SHA-256。
- 同名且内容一致的目标文件跳过并恢复时间属性。
- 同名但内容不同的文件写入目标根下的 `冲突文件/`。
- Windows不支持的文件名和大小写冲突会跳过并写入报告。
- 只建立选中文件所需的父目录，不创建其他空目录。
- 每次恢复生成 HTML 和 JSON 两份报告。

Windows程序启动后只监听 `127.0.0.1` 并打开默认浏览器。它不会连接或下载 WebDAV，只读取用户已经下载好的完整任务目录。

NAS后台的“从已下载的数据块恢复”也不连接 WebDAV。所选目录必须保留任务根下的 `objects/` 路径；可以只放入 `.backup-plan` 列出的块，缺块文件会单独报告。

管理API会在服务层校验路径：备份源必须位于 `WCB_SOURCE_ROOT`，恢复输出必须位于 `WCB_RESTORE_ROOT`；手动下载的任务目录可位于这两个映射中的任意一个。

## 从零接管远端任务

在“从远端恢复任务”中添加 WebDAV备份根，程序会扫描一级任务目录并读取双份公开描述。输入任务密码后：

1. 解密双份任务索引和每个版本索引。
2. 重建 SQLite、本地索引、版本、哈希和任务配置。
3. 优先选择最新完整版本，检查源目录中每个相对路径是否存在且大小一致；额外新文件不影响匹配。
4. 检查所有引用远端对象的名称和大小。
5. 完全匹配后恢复写入；任何差异都会保持只读并明确列出。

修改 WebDAV IP、根路径或凭据也走相同的永久 UUID 只读验证流程。UUID不一致时拒绝写入。

## 安全边界

- 任务密码不会写入远端；NAS本地 SQLite明文保存任务密码以支持无人值守运行。
- WebDAV凭据也仅保存在 NAS本地。使用纯HTTP时，HTTP Basic凭据可被局域网监听，因此只适用于可信局域网内的WebDAV转换软件。
- 任务名、任务UUID、格式版本、KDF参数和盐是远端公开信息；原文件名、路径、内容和完整索引均加密。
- Web管理后台没有鉴权，任何能访问端口的人都能浏览已解锁的本地索引并操作任务。
- 加密可以阻止网盘直接识别媒体内容，但不隐藏对象数量、大小和上传时间。

详细格式见 [docs/storage-format.md](docs/storage-format.md)，内部接口见 [docs/api.md](docs/api.md)。

## 本地开发与测试

本项目固定使用 Go 1.26.5、Node.js 24.15.0，并提交 `go.sum` 和 `package-lock.json`。

```bash
cd web
npm ci
npm run lint
npm run format:check
npm test
npm run build

cd ..
gofmt -w cmd internal
go vet ./cmd/... ./internal/...
go test -race ./cmd/... ./internal/...
```

不要在本地运行 Docker 测试。容器构建、启动和健康检查只在 GitHub Actions 中执行。
