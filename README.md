# WebDAV Cold Backup

面向群晖 NAS 冷备份和异地转移的 WebDAV 分块备份工具。NAS 端通过 Docker Compose 运行，提供简体中文管理后台；Windows 10/11 x64 可使用单文件离线恢复程序。

> 当前格式不提供冗余或纠删码。永久丢失一个数据块会导致该块包含的文件无法完整恢复，但不会连带损坏其他数据块。任务密码丢失后无法恢复数据。

## 已实现功能

### 备份任务

- 一个任务可选择一个或多个互不嵌套的 NAS 目录。
- 支持“版本快照”和“持续归档”两种固定模式。
- 支持 1 GB、2 GB、3.7 GB 三种远端对象容量上限。
- 大文件自动切分，小文件自动装包；大文件尾块不会与其他文件混装。
- 文件移动或重命名后，快照模式可复用已有远端数据块。
- 源文件删除不会立即删除旧版本数据。
- 多任务使用串行队列，单个任务可设置 1—3 个对象并发上传。

### 自动运行与传输

- 支持手动、每天固定时间、每周固定星期和时间运行。
- 错过的计划不会在 NAS 重新启动后补执行。
- 支持分别设置上传和下载限速。
- 任务详情实时显示扫描、哈希、加密上传和索引发布阶段，以及文件、对象和字节进度。
- 支持暂停和继续；暂停会等待当前对象处理完成。
- 上传失败先快速重试一次，再按 1、5、25 分钟退避重试。
- 未完成对象会删除并从头上传，不保留无法确认的半成品。
- WebDAV 地址、根目录、用户名或密码改变后，可重新验证并接管同一任务。
- WebDAV 目标可预存在 NAS 本地，创建任务和远端恢复均可直接复用。
- 新目标按“填写地址和凭据、浏览选择远端目录、按需预存”的顺序配置。

### 版本与完整性管理

- 快照模式默认保留最近 3 个版本，也可锁定为永久版本。
- 完全没有变化时不会创建空版本。
- 部分文件失败时仍可发布明确标记的“不完整版本”。
- 快速检查只读取远端对象名称和大小，不下载数据块。
- 缺失、大小不符和未引用对象都会在后台单独显示。
- 清理未引用对象、删除任务或删除版本均有确认保护。
- NAS 本地数据库丢失后，可通过远端双份索引重建任务。

### 恢复

- 管理后台可直接从 WebDAV 恢复整个任务、目录或多个文件。
- 可导出 `.backup-plan`，只下载选中文件需要的数据块。
- NAS 后台也可从已经手动下载的任务目录离线恢复。
- Windows x64 恢复端完全离线，不负责下载 WebDAV 数据。
- 默认选择最新完整版本，也允许手动选择其他或不完整版本。
- 所有恢复入口都会在写入完成后重新计算原文件 SHA-256；只有核验通过才发布最终文件，并恢复创建时间和修改时间。
- 已存在且内容相同的文件会跳过；内容不同的文件进入 `冲突文件/`。
- 缺块只影响对应文件，其他完整文件仍可恢复。
- 每次恢复生成 HTML 和 JSON 报告。

### 隐私与自恢复

- 原文件名、目录、文件内容和完整索引均经过加密。
- 远端只显示任务名、随机目录、随机对象名和 `part-*` 分块序号。
- 每个数据对象带有加密自恢复头；远端索引全部丢失时仍可尽力抢救现有块。
- 管理后台不提供登录鉴权，适合只在可信局域网使用。

## 群晖部署

项目提供的 `docker-compose.yml` 已经使用固定值，不依赖 `.env`：

- 管理端口：`19001`
- 容器用户：`1026:100`
- 配置目录：`/volume1/docker/webdav-cold-backup/config`
- 缓存目录：`/volume1/docker/webdav-cold-backup/cache`
- 备份源：整个 `/volume1`，在容器内只读映射为 `/sources`
- 恢复目录：`/volume1/docker/webdav-cold-backup/restore`

先在群晖上创建目录：

```bash
mkdir -p /volume1/docker/webdav-cold-backup/config
mkdir -p /volume1/docker/webdav-cold-backup/cache
mkdir -p /volume1/docker/webdav-cold-backup/restore
```

然后启动：

```bash
docker compose up -d
```

管理后台地址：

```text
http://群晖IP:19001
```

使用的公开镜像为：

```text
ghcr.io/ccawmiku/webdav-cold-backup:v1.2.0
```

项目不会发布 `latest` 标签。

### 群晖容器用户与目录权限

当前 Compose 按这台群晖管理员账号的实际身份固定使用 `1026:100`。UID 或 GID 与实际身份不一致时，某些目录会在扫描时返回 `permission denied`。

修改 Compose 后需要重新创建容器，使新的用户身份生效：

```bash
docker compose up -d --force-recreate
```

如果重新创建后仍只有某个目录无法读取，说明该目录自身的 ACL 或父目录遍历权限没有授给 `1026:100`，需要在 DSM 中为该账号或用户组补充读取权限。`/sources` 使用 `:ro` 挂载，程序不能通过这个挂载修改或删除源文件。

不需要创建 `.env` 文件。如果数据不在 `/volume1`，直接修改 Compose 的宿主机目录即可。

## 使用流程

### 创建第一次备份

1. 打开 `http://群晖IP:19001`。
2. 创建任务并选择“版本快照”或“持续归档”。任务模式、密码和对象容量创建后不能修改。
3. 从 `/sources` 下选择一个或多个源目录，并为每个源设置恢复根名称。
4. 选择预存 WebDAV 目标；或先填写地址、用户名和密码，再浏览选择远端目录并按需预存。
5. 选择手动、每天或每周计划，然后执行任务。

程序会用小型临时对象测试 WebDAV 的 `MKCOL`、`PUT`、`PROPFIND`、`MOVE` 和 `DELETE`。10 分钟内仍在修改的文件会静默跳过；符号链接、回收站及常见 NAS 系统目录会忽略并计入运行摘要。

默认忽略的目录包括 `#recycle`、`@Recycle`、`$RECYCLE.BIN`、`.Trash` 和 `@eaDir`。任何源目录权限不足时，本次扫描停止，不发布新版本。

### 从 WebDAV 恢复

在任务的“恢复”页面选择版本和文件，再选择 `/restore` 下的目标目录。系统先确认所需远端对象名称和大小，随后下载、解密并校验文件。

### 手动下载后恢复

1. 在管理后台选择文件并导出 `.backup-plan`。
2. 根据计划下载所需数据块，并保留任务根下的 `objects/` 目录结构。
3. 将任务目录放入 `/sources` 或 `/restore` 映射范围。
4. 在“从已下载的数据块恢复”中选择该目录和恢复目标。

缺少数据块的文件会单独报告，不影响其他完整文件。

### Windows 离线恢复

从 GitHub Actions 制品下载 `webdav-cold-backup-restore-windows-amd64.exe`。运行后程序只监听随机的 `127.0.0.1` 端口，并打开本地网页：

1. 选择已经完整下载的任务目录。
2. 输入任务密码。
3. 选择版本和文件。
4. 选择本地恢复目录并开始恢复。

Windows 程序不连接 WebDAV，也不会自动下载任何数据。恢复页面会实时显示预检对象、解密写入、恢复字节、当前对象、当前文件和 SHA-256 核验进度；核验不通过的临时文件不会写成最终文件。

## 具体技术实现

### 分块与装包

最终密文对象上限固定为：

- `1,000,000,000` 字节
- `2,000,000,000` 字节
- `3,700,000,000` 字节

每个对象预留 16 MiB 用于加密头、元数据和认证标签。文件大于有效容量 80% 时独立存放；超过容量时连续切分，最后的尾块也不会和其他文件混装。小文件按源根、相邻目录和文件名排序装包，单包最多 5000 个文件。

同一大文件的所有分块放在同一个随机文件组目录中：

```text
任务名/
  objects/
    3f/
      <随机文件组ID>/
        part-00001-<随机对象ID>.wcb
        part-00002-<随机对象ID>.wcb
```

### 加密格式

- 密钥派生：Argon2id，时间成本 3、内存 64 MiB、并行度 2、输出 256 bit。
- 内容加密：XChaCha20-Poly1305。
- 加密帧：每帧 4 MiB，独立认证。
- Nonce：随机 16 字节前缀加 64 位帧序号。
- 关联数据：公开头 SHA-256 加帧序号。

使用的是公开、成熟的 `golang.org/x/crypto` 实现，没有自制加密算法。任务密码不会写入远端，但会明文保存在 NAS 本地 SQLite 中，以支持无人值守备份。预存 WebDAV 目标的密码同样只保存在 NAS 本地，目标列表 API 不会回传密码。

### 索引与远端布局

```text
任务名/
  task-a.json
  task-b.json
  catalog-a.wci
  catalog-b.wci
  objects/
  snapshots/
    <版本ID>/
      index-a.wci
      index-b.wci
      COMPLETE
      KEEP
```

`task-a.json` 和 `task-b.json` 是公开启动描述，只包含格式版本、任务 UUID、任务名、Argon2id 盐和时间，不包含密码、源路径或文件名。任务目录内索引和版本索引均加密并保存双份。

数据对象的加密自恢复头记录文件 ID、首次路径、大小、SHA-256、时间、对象偏移和大文件分片位置。当全部索引丢失时，可以扫描现有 `.wcb` 对象重建可恢复文件；但无法证明是否有整个对象被删除，也不能找回索引丢失前发生的后续路径移动。

详细格式见 [docs/storage-format.md](docs/storage-format.md)，HTTP 接口见 [docs/api.md](docs/api.md)。

### 快照模式

- 扫描结果有变化时才发布新版本。
- 未修改文件、移动和重命名可跨版本复用对象。
- 同一版本中复制出的新文件按独立文件处理。
- 源端删除只影响新版本。
- 最近 N 个未锁定版本之外的数据立即进行引用检查和垃圾回收。
- `KEEP` 是非空 JSON 锁定文件；永久版本不计入最近 N 个普通版本。

### 归档模式

- 每个内容哈希只在首次出现时上传。
- 记录内容第一次出现的路径。
- 后续移动、重命名、复制和源端删除都不修改已有记录。
- 手动删除文件记录时，只有不再被任何文件引用的数据对象才会删除。
- 如果源文件仍在 NAS，下次扫描会重新加入，不设置永久排除项。

### WebDAV 发布与检查

对象先上传到随机 `.partial-*` 路径。程序通过 `PROPFIND` 确认远端大小后，再用 `MOVE` 发布正式对象名。失败时删除临时对象并从头重传。

快速完整性检查只读取远端名称和大小，不下载数据，也不依赖不稳定的 ETag。真正恢复时，每个加密帧必须通过 AEAD 认证，完整文件还必须通过备份索引中的 SHA-256 校验。

### 从零接管

从新的 NAS 或空数据库接管远端任务时，程序会：

1. 扫描 WebDAV 根目录下一级任务目录。
2. 读取双份任务描述并核对永久 UUID。
3. 输入任务密码后解密任务和版本索引。
4. 重建 SQLite、本地文件索引、版本和任务设置。
5. 优先检查最新完整版本与当前映射源路径、文件大小是否一致。
6. 检查全部引用对象的名称和大小。
7. 完全匹配后允许继续写入；存在差异时保持只读。

修改 WebDAV IP、根路径或凭据也使用相同的 UUID 与只读验证流程。

### 恢复冲突处理

- 同名且 SHA-256 相同：跳过写入并恢复时间属性。
- 同名但内容不同：写入恢复根下的 `冲突文件/`。
- Windows 非法文件名或大小写冲突：跳过并写入报告。
- 未选择的空目录：不会创建。
- 缺少部分块：只把对应文件标记为不可恢复。

### 本地目录和安全边界

- `/config`：SQLite、本地任务状态和运行记录。
- `/cache`：上传前的临时加密对象，空间需求约为对象容量乘上传并发数。
- `/sources`：只读备份源。
- `/restore`：NAS 恢复输出和可选的手动下载任务目录。

服务层强制要求备份源位于 `/sources`，恢复输出位于 `/restore`。手动下载的任务目录可以位于 `/sources` 或 `/restore`。

管理后台没有鉴权，任何能访问 `19001` 端口的人都能浏览本地索引并操作任务。HTTP WebDAV Basic 凭据也可能被同网段监听，因此服务只应部署在可信局域网。

## 本地开发与测试

项目固定使用 Go 1.26.5 和 Node.js 24.15.0，并提交 `go.sum` 与 `package-lock.json`。

```bash
cd web
npm ci
npm run lint
npm run format:check
npm test
npm run build

cd ..
gofmt -w cmd internal
go vet ./cmd/... ./internal/...
go test -race ./cmd/... ./internal/...
```
